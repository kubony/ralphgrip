# Agent Issue Workflow Spec

> 생성일: 2026-03-16
> 목적: issue 타입 프로젝트의 기본 상태와 오케스트레이터/에이전트 운영 규칙을 고정한다.

## 적용 범위

- 이 문서는 **issue 타입 프로젝트**에 적용한다.
- 새 issue 프로젝트 생성 시 기본 상태는 아래 순서로 생성한다.
  - `Open`
  - `Todo`
  - `In Progress`
  - `Issue`
  - `Resolved`
  - `Closed`

## 상태 의미

| 상태 | 주체 | 의미 |
|------|------|------|
| Open | 오케스트레이터 | 계획 단계에서 오케스트레이터가 생성한 상태 |
| Todo | 오케스트레이터/시스템 | 에이전트가 할당되어 실행 대기 중인 상태 |
| In Progress | 에이전트 | 에이전트가 실제 작업을 시작한 상태 |
| Issue | 에이전트 | 진행 중 문제, 블로커, 추가 판단 필요 상황이 발생한 상태 |
| Resolved | 에이전트 | 에이전트가 작업 완료라고 판단한 상태 |
| Closed | 오케스트레이터 | 결과물을 점검하고 최종 완료 승인한 상태 |

## 권장 상태 전이

```text
Open → Todo → In Progress → Resolved → Closed
```

예외 흐름:

```text
In Progress → Issue
Issue → Todo | In Progress | Resolved
Resolved → In Progress
```

## 운영 규칙

1. 오케스트레이터는 계획 단계 산출물을 바탕으로 issue를 `Open`으로 만든다.
2. 실행할 에이전트가 정해지면 상태를 `Todo`로 이동한다.
3. 에이전트가 실제 작업을 시작하면 `In Progress`로 이동한다.
4. 작업 중 문제가 발생하면 `Issue`로 이동한다.
5. 에이전트가 작업 완료로 판단하면 `Resolved`로 이동한다.
6. 오케스트레이터가 산출물을 검수하고 승인하면 `Closed`로 이동한다.
7. `Closed`만 최종 완료 상태로 간주한다.

## RalphGrip / RalphGrip E2E 원칙

RalphGrip(RalphGrip)은 단순 추적 도구가 아니라, 프로젝트 생성부터 최종 결과물 완성까지 전체 실행 흐름을 다루는 시스템이어야 한다.

### 대표 E2E 시나리오

1. 영상 제작 프로젝트를 생성한다.
2. 오케스트레이터가 기획/스크립트/제작/편집/검수 태스크를 만든다.
3. AI 에이전트들을 스폰한다.
4. 각 태스크에 에이전트를 할당한다.
5. 에이전트가 실제 작업을 수행하고 상태를 갱신한다.
6. 오케스트레이터가 결과물을 검수하고 재작업을 조정한다.
7. 최종 영상 결과물을 완성한다.

즉, RalphGrip의 목표는 **모든 핵심 기능을 활용해 AI 에이전트가 실제로 일하고 최종 산출물까지 만드는 E2E 운영**을 지원하는 것이다.
