# 전체 프로젝트 진행현황 페이지 Spec

> 생성일: 2026-03-02
> 인터뷰 기반 자동 생성

## Background

유태준 대표가 AgentGrip 주행 부문에 ~30억 투자 대비 성과 부진을 지적하며, **프로젝트별 현황을 투명하게 보고**할 것을 요구. 박유빈 총괄과 협의하여 일회성 장표가 아닌 **AgentGrip 소프트웨어에 정식 탑재**하기로 결정.

- 내일(3/3) PRD 발표에 바로 사용 + 이후 상시 관리 도구
- AgentGrip는 프로젝트 진행 관리 담당

## Goals

- 모든 프로젝트의 진행 현황을 **한 화면에** 시각화 (크로스-프로젝트 Gantt)
- 프로젝트별 사업 요약 정보 (진행률, 인원, 상태, 이슈, 간략 금액) 표시
- 경영진이 한눈에 파악할 수 있는 대시보드 수준의 가독성
- 3/3 발표에서 실제 5개 프로젝트 데이터로 라이브 데모

## Users

- **1차**: 유태준 대표 (경영진 보고용) - 핵심 숫자와 상태를 한눈에
- **2차**: 서인근, 박유빈 (PM/팀장) - 프로젝트 관리 및 보고 준비
- **향후**: 전사 확산 시 다른 사업부 리더

## Requirements

### Must Have

- **크로스-프로젝트 Gantt 차트**
  - 모든 프로젝트를 하나의 타임라인에 표시
  - 프로젝트별 기간 (시작일 ~ 종료일) 바 시각화
  - 프로젝트 상태별 색상 구분 (영업중/수행중/완료 등)
  - 기존 AgentGrip 타임라인 컴포넌트 활용 가능성 검토
- **프로젝트 사업 요약 카드/리스트**
  - 프로젝트별: 상태, 진행률(완료 work item 비율), 투입 인원 수, 기간
  - 간략 금액: 사업 규모 (예: 5억)
  - 주요 이슈/지연 항목 수
- **헤더 내비게이션에 새 메뉴** 추가
  - My Work, Projects와 나란히 "사업현황" 또는 "Pipeline" 메뉴
  - 별도 라우트: `/pipeline` 또는 `/business`
- **프로젝트 settings JSONB에 사업 필드 추가**
  - `business_amount`: 사업 규모 (숫자, 원 단위)
  - `business_phase`: 사업 단계 (prospect/contracted/active/closed)
  - `business_start_date`, `business_end_date`: 사업 기간
  - `business_note`: 간략 메모
- **프로젝트 설정 UI에서 사업 필드 편집 가능**
  - 사용자가 직접 UI에서 데이터 입력

### Nice to Have

- Gantt 바 클릭 시 해당 프로젝트로 이동
- 사업 단계별 필터링/그룹핑
- 프로젝트 요약 카드에서 최근 활동 미리보기
- 인쇄/PDF 내보내기 (발표 백업용)
### Not Doing

- 수주 확도 관리
- 현금 회수 추적 (계약금/중도금/잔금)
- 인력 수급 매칭 (수요 vs 공급)
- 영업 파이프라인 상세

- 품의(구매요청) 프로세스 → 별도 시스템

## Technical Constraints

- **기술 스택**: Next.js 16 + React 19 + Supabase + Tailwind + shadcn/ui
- **기존 타임라인 활용**: `timeline-*.tsx` 컴포넌트 참고하되, 크로스-프로젝트용으로 단순화 필요
  - 기존 타임라인: work item 단위 → 새 페이지: project 단위
  - 기존 Gantt 복잡도(드래그, 가상화 등) 불필요, 읽기 전용으로 충분
- **DB 변경**: settings JSONB 확장으로 마이그레이션 불필요
- **데이터 소스**: `projects` 테이블 + `work_items` (진행률 계산) + `project_members` (인원 수)
- **성능**: 프로젝트 수가 수십 개 수준이므로 가상화 불필요

## Page Structure (안)

```
/pipeline (또는 /business)
├── 상단: 요약 통계 카드 (전체 프로젝트 수, 수행중, 영업중, 완료)
├── 중앙: 크로스-프로젝트 Gantt 차트
│   ├── Y축: 프로젝트명 (+ 사업규모, 상태 배지)
│   ├── X축: 타임라인 (월 단위)
│   └── 바: 사업 기간, 상태별 색상
└── 하단 또는 사이드: 프로젝트별 요약 카드 (클릭 시 상세)
```

## Open Questions

- 헤더 메뉴명: "사업현황" vs "Pipeline" vs "전체현황"?
- Gantt에 work item 레벨 마일스톤도 표시할 것인가? (예: 긴트 잔금 회수 시점)
- 프로젝트 순서: 사업 단계별 그룹핑 vs 시작일순 vs 수동 정렬?

## Key Decisions

- **역할**: Gantt + 사업요약은 AgentGrip에서 관리
- **DB 확장 방식**: settings JSONB 활용 (마이그레이션 없이 유연하게)
- **UI 위치**: 헤더 내비게이션에 별도 메뉴
- **데이터 입력**: 사용자가 프로젝트 설정에서 직접 입력 (seed 불필요)
- **Gantt 구현**: 기존 타임라인 참고하되 읽기 전용으로 단순화
